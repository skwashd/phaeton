# Modify Cdk Writer Picofun Construct

**Priority:** P1
**Effort:** M
**Gap Analysis Ref:** Item #12

## Overview

The CDK writer (at `packager/src/n8n_to_sfn_packager/writers/cdk_writer.py`) generates `stacks/workflow_stack.py` with Lambda function constructs, Lambda layer constructs, and workflow infrastructure. Currently, `PICOFUN_API_CLIENT` functions are treated identically to regular Python Lambda functions — they get individual `lambda_.Function` constructs with standard bundling.

With PicoFun integration, these functions should instead be handled by a PicoFun CDK construct (generated by `CdkGenerator` in TASK-0010). This requires four changes:

1. **Skip PicoFun functions** in `_wf_lambda_functions()` (line 478)
2. **Add PicoFun construct import** in `_wf_imports()` (line 301)
3. **New `_wf_picofun_construct()` section** to instantiate the PicoFun construct
4. **Reference PicoFun layer** in `_wf_lambda_layers()` (line 426) for PicoFun functions

## Dependencies

- **Blocked by:** TASK-0011 (PicoFunWriter must be integrated so PicoFunOutput is available)
- **Blocks:** TASK-0019

## Acceptance Criteria

1. `_wf_lambda_functions()` skips functions with `function_type == LambdaFunctionType.PICOFUN_API_CLIENT`.
2. When PicoFun functions exist, `_wf_imports()` includes `from construct import PicoFunConstruct`.
3. A new `_wf_picofun_construct()` method generates code to instantiate the PicoFun CDK construct.
4. The PicoFun construct is wired to VPC/security group if VPC config is present.
5. `_write_workflow_stack()` accepts optional PicoFun metadata and includes the picofun construct section.
6. When no PicoFun functions exist, the workflow stack is generated identically to before (no PicoFun imports or sections).
7. `uv run pytest` passes in `packager/`.
8. `uv run ruff check` passes in `packager/`.

## Implementation Details

### Files to Modify

- `packager/src/n8n_to_sfn_packager/writers/cdk_writer.py`

### Technical Approach

1. In `_wf_lambda_functions()` (line 478), add a filter at the top:
   ```python
   non_picofun = [s for s in specs if s.function_type != LambdaFunctionType.PICOFUN_API_CLIENT]
   ```
   Use `non_picofun` instead of `specs` for the loop that generates individual `lambda_.Function` constructs.

2. In `_wf_imports()` (line 301), add conditional import when PicoFun functions are present:
   ```python
   if has_picofun:
       lines.append("from construct import PicoFunConstruct")
   ```

3. Create `_wf_picofun_construct()` method:
   ```python
   def _wf_picofun_construct(self, picofun_output: PicoFunOutput, ...) -> str:
       """Generate code to instantiate the PicoFun CDK construct."""
   ```
   The generated code:
   - Instantiates `PicoFunConstruct(self, "PicoFun", ...)` with the layer path and function specs
   - If VPC config is present, passes `vpc=shared_stack.vpc`, `security_groups=[shared_stack.lambda_sg]`
   - Connects PicoFun Lambda functions to the state machine via `lambda_functions` dict

4. In `_write_workflow_stack()` (line 263), accept an optional `picofun_output: PicoFunOutput | None` parameter. When present, include `_wf_picofun_construct()` in the `sections` list after layers but before the state machine section.

5. In `_wf_lambda_layers()` (line 426), if PicoFun output exists, add the PicoFun layer reference for PicoFun functions.
